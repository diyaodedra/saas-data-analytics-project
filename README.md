# SaaS Customer Churn Analysis

## Overview

This project analyzes customer behavior for a SaaS company to understand patterns related to churn, engagement, and revenue. The analysis focuses on identifying factors that influence customer retention and business performance.

The project follows a typical data analysis workflow using Excel and SQL.

## Tools Used

* Excel – Data cleaning, pivot tables, and data visualization
* MySQL / SQL – Data querying and analysis

## Dataset

The dataset contains information about 2,800 SaaS users, including:

* Subscription plan type
* Monthly subscription fee
* Engagement level
* Support tickets
* Payment failures
* Customer tenure
* Last login activity
* Churn status

## Analysis Conducted

The following analyses were performed using pivot tables and charts:

* Churn by subscription plan
* Engagement level vs churn
* Support tickets vs churn
* Payment failures vs churn
* Customer tenure vs churn
* Active vs inactive users vs churn
* Last login activity vs churn
* Revenue by subscription plan
* Average revenue per user by churn

## Insights

Key insights and interpretations from the charts are documented in the file:

**Graph.insights.pdf**

This document explains the patterns observed in the visualizations and highlights the main drivers of customer churn and revenue.

## Project Purpose

The purpose of this project is to demonstrate practical data analysis skills, including:

* Data exploration
* Pivot table analysis
* Data visualization
* Business insight generation

This type of analysis helps SaaS companies better understand customer behavior and identify opportunities to improve retention.
