# SaaS Customer Churn Analysis

## Project Overview

This project analyzes customer churn and usage patterns for a SaaS startup dataset.

The goal is to identify factors that contribute to churn and understand user engagement behavior.

## Tools Used

* Excel (data cleaning and pivot table analysis)
* MySQL / phpMyAdmin (data querying and analysis)

## Dataset

The dataset contains customer subscription and usage data including:

* plan type
* monthly fees
* engagement levels
* support tickets
* payment failures
* tenure
* churn status

## Key Analysis

* Revenue by subscription plan
* Churn rate by tenure group
* Engagement level vs churn
* Payment failures vs churn
* Support tickets vs churn

## Key Insights

* Customers with low engagement churn significantly more often.
* Higher payment failures are associated with increased churn risk.
* Longer-tenure customers are less likely to churn.
